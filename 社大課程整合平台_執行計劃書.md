

**社大課程整合平台**

*Community Course Aggregator*

**產品執行計劃書  v1.0**

2026 年 6 月 ｜ Side Project

# **1\. 專案概述**

## **1.1 背景與動機**

台北市各社區大學（南港、文山、松山等）均採用同一套 frog.tw 課務系統，但各校只有單校查詢介面。使用者若想橫向比較多校課程，必須分開瀏覽多個網站，且現有 UI 在行動裝置上體驗不佳。

本 Side Project 目標：打造一個跨校課程整合瀏覽平台，讓使用者能夠搜尋、比較、收藏來自多所社大的課程，並以週曆視圖管理報名意向。

## **1.2 目標用戶**

* 有意報名社大課程、想橫向比較多校選擇的一般市民

* 希望在報名截止前整理「候選清單」的使用者

* 在意上課時段衝突、需要確認行程的職場人士

## **1.3 核心價值主張**

* 一站式瀏覽多校課程，不再跳頁

* 強大篩選：時段、星期、類別、學校、折扣、開課狀態

* 課程比較：最多 3 門並排，快速決策

* 我的最愛 \+ 衝堂偵測：幫你管理候選清單

* 週曆視圖：直覺看出課程在哪些時間

## **1.4 範疇限制（Scope）**

* 不提供實際報名功能（連結至原網站即可）

* 不儲存個人帳號資料（純前端 localStorage）

* 資料每週更新一次（非即時）；額滿狀態僅供參考

* 初版支援台北市 3～5 所社大，未來可擴充

# **3\. 頁面規劃**

共 6 個頁面（含 Modal 層），以 React Router v7 管理路由。所有路由均為 SPA client-side routing，配合 GitHub Pages 的 404 redirect trick 處理重新整理。

| 頁面名稱 | 路由 | 主要功能 | 備註 |
| :---- | :---- | :---- | :---- |
| **首頁 / 課程列表** | //courses | 課程卡片、篩選列、搜尋、分頁或虛擬捲動 | 主視覺 |
| **課程詳細 Modal** | （overlay） | 學費、老師、時段、地點、名額、加入最愛鈕 |  |
| **跨校比較頁** | /compare | 左右並排最多 3 欄，差異高亮 | 次要 |
| **我的最愛** | /favorites | 收藏清單、衝堂偵測、週曆視圖切換 | 次要 |
| **時間表頁** | /schedule | FullCalendar 週視圖，顯示已收藏課程 | 次要 |
| **關於 / 資料來源** | /about | 說明資料來源、更新頻率、免責聲明 |  |

## **3.1 課程列表頁（主頁）— 詳細說明**

* 課程卡片：縮圖、課程名稱、學校、老師、星期+時段、費用、狀態 Badge

* 篩選列（Sidebar 或 Drawer）：多選星期、時段 toggle、類別 tree、學校 checkbox、折扣、狀態

* 搜尋框：即時 filter 課程名稱 / 老師名（client-side，無需 API）

* 排序：依開課日、費用升冪/降冪、學校

* 虛擬捲動（react-virtual）：230+ 筆資料不影響效能

## **3.2 跨校比較頁 — 詳細說明**

* 使用者在課程卡片勾選「加入比較」（最多 3 門）

* 比較 Table：每欄一門課，列出所有 Schema 欄位，差異欄位高亮顯示

* 底部顯示「前往報名」連結（跳至原校官網）

## **3.3 我的最愛 — 詳細說明**

* 卡片式清單，可拖曳排序優先順序（dnd-kit）

* 衝堂偵測：同星期同時段有 2 門以上課程，自動標示紅框警告

* 匯出功能：複製清單文字 / 截圖分享

# **4\. 技術架構**

| 層級 | 技術 | 選擇原因 |
| :---- | :---- | :---- |
| **前端框架** | **React \+ Vite** | 熟悉的技術棧，極快的 HMR，build 後為純靜態 |
| **路由** | **React Router v7** | 已在創宇專案使用，無學習成本 |
| **UI 元件** | **shadcn/ui \+ Tailwind** | 已在創宇使用；快速搭出乾淨 UI |
| **狀態管理** | **Zustand** | 輕量，管理篩選條件 / 收藏 / 比較清單 |
| **資料查詢** | **TanStack Query** | 已熟悉；快取 JSON 請求、loading/error state |
| **日曆元件** | **FullCalendar (React)** | 週視圖顯示收藏課程，支援自訂 event render |
| **持久化** | **localStorage** | 我的最愛、比較清單；無需後端帳號 |
| **爬蟲** | **Node.js \+ cheerio** | 輕量 HTML parser，適合解析 server-side render 頁 |
| **排程** | **GitHub Actions (cron)** | 每週一 00:00 觸發爬蟲，結果 commit 進 repo |
| **資料存放** | **GitHub repo JSON** | courses/{school}.json，由 jsDelivr CDN 加速讀取 |
| **部署** | **GitHub Pages** | Vite build → gh-pages，免費靜態托管 |
| **型別** | **TypeScript** | 課程 Schema interface，避免 JSON 欄位錯誤 |

## **4.1 前端架構圖（文字描述）**

App → React Router v7 → 各頁面 Component

　　└ TanStack Query → fetch JSON from jsDelivr CDN

　　└ Zustand Store → filterState / favorites / compareList

　　└ shadcn/ui \+ Tailwind → UI 元件

　　└ FullCalendar → 週視圖

## **4.2 資料流架構**

GitHub Actions（週一 cron）

  → Node.js 爬蟲（cheerio）

  → 解析 frog.tw HTML

  → 輸出 data/courses/{school}.json

  → git commit & push

  → jsDelivr CDN 自動快取

  → React app fetch → TanStack Query 快取 → Zustand filter → UI render

# **5\. 資料設計**

## **5.1 Course Schema（TypeScript Interface）**

| 欄位名稱 | 型別 | 說明 |
| :---- | :---- | :---- |
| **id** | *string* | hash from URL query param u=xxxxx |
| **school** | *string* | "nangang" | "wenshan" | … |
| **name** | *string* | 課程名稱（完整） |
| **teacher** | *string\[\]* | 授課老師（可多人） |
| **category** | *string* | "生活藝能－烹飪美食類" … |
| **dayOfWeek** | *number* | 0=日、1=一 … 6=六 |
| **timeSlot** | *"morning"|"afternoon"|"evening"* | 上午、下午、晚上 |
| **startDate** | *string (ISO)* | 開課日期 |
| **location** | *string* | 地點完整名稱 |
| **fee** | *number* | 學分費（元），0 \= 免費 |
| **discount** | *string?* | "八折"|"五折"|null … |
| **status** | *string* | "open"|"full"|"cancelled"|"delayed" |
| **weeks** | *number?* | 總週數（若有） |
| **isNew** | *boolean* | 新課程標示 |
| **isMixed** | *boolean* | 混成（線上+實體） |
| **detailUrl** | *string* | 原始課程詳細頁 URL |
| **imageUrl** | *string?* | 課程縮圖（選填） |

## **5.2 資料存放策略**

資料全部以靜態 JSON 存放於 GitHub repo，不需要任何後端服務：

| 資料類型 | 存放位置 | 說明 |
| :---- | :---- | :---- |
| **課程資料** | data/courses/\*.json | 每校一個 JSON，由爬蟲每週覆寫 |
| **我的最愛** | localStorage | key: "cca\_favorites"，儲存 course id array |
| **比較清單** | Zustand（記憶體） | 最多 3 筆，重新整理後清空 |
| **使用者偏好** | localStorage | 記憶最後一次的篩選條件、深色模式 |

## **5.3 版權與合規說明**

* 抓取對象為政府補助的公共教育機構，課程資訊屬公共性質

* 只存「開課事實」欄位，不複製課程介紹文字或老師照片

* UI 上清楚標示「資料來源：各校課務系統」並附原始連結

* 用戶點擊報名時導向原始官網，本平台不介入任何金流

* 若收到任何學校要求下架，立即配合移除該校資料

# **7\. 風險評估**

| 風險項目 | 影響程度 | 對應策略 |
| :---- | :---- | :---- |
| frog.tw HTML 結構改版 | **高** | 爬蟲加入欄位驗證；GitHub Actions 失敗時發 Issue 通知 |
| 各校 URL 規則不一致 | 中 | 逐校手動驗證後再加入爬蟲列表 |
| 學校要求移除資料 | 低 | 配置 per-school 開關，可快速下架單一學校資料 |
| localStorage 容量上限 | 低 | 只存 id array，不存完整課程物件；預估遠低於 5MB 限制 |
| GitHub Pages SEO / 分享連結 | 低 | 加入 og:title / description meta；分享連結帶 query string filter |
