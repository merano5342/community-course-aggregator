# 全域導覽列與應用程式佈局

---

**頁面位置**：全站共用（所有路由）  
**頁面路由**：所有路由共用 Layout  
**所屬功能模組**：0.0 全局功能  
**最後更新**：2026-06-20（v1.1：行動版改為底部導覽列，新增首頁路由）

---

## 需求背景

社大課程整合平台為 SPA，所有頁面共用頂層 Layout 元件。導覽結構需同時服務桌面與行動使用者，並即時反映「比較清單數量」與「最愛數量」等全域狀態。行動版採固定底部導覽列，避免漢堡 Drawer 的多步操作。

---

## 佈局結構

### 桌面版（≥ 768px）

```
App
├── <AppSidebar>      深色圖示側邊欄（固定左側，var(--sidebar-width)）
│   ├── Logo
│   ├── 5 個導覽圖示（home / courses / saved / schedule / compare）
│   └── 底部資訊圖示（/about）
└── <main>
    ├── <Header>      置頂導覽列（含品牌名 + 水平選單 + 徽章）
    ├── <Outlet>      頁面內容區
    ├── <CompareBar>  底部固定浮動列（bottomOffset=0）
    └── <Toaster>     全域 Toast 通知
```

### 行動版（< 768px）

```
App
└── <main>
    ├── <Header>      置頂（僅顯示 Logo，隱藏水平選單）
    ├── <Outlet>      頁面內容區
    ├── <CompareBar>  底部固定浮動列（bottomOffset=60，顯示在底部導覽列上方）
    └── <BottomNav>   固定底部導覽列（高度 60px，含 5 個圖示 + 標籤 + 徽章）
```

當前路由對應的圖示以 `var(--color-accent)` 高亮。

## 行動版底部導覽列（BottomNav）

固定於螢幕底部，高度 60px，深色背景（`var(--color-text-primary)`）。

| 位置 | 項目 | 圖示 | 路由 | 徽章 |
|------|------|------|------|------|
| 1 | 首頁 | house | `/` | — |
| 2 | 課程 | list | `/courses` | — |
| 3 | 最愛 | bookmark | `/favorites` | 收藏數量 > 0 時顯示 |
| 4 | 時間表 | calendar | `/schedule` | — |
| 5 | 比較 | bar-chart | `/compare` | 已選數量 > 0 時顯示 |

- 每個項目含圖示 + 文字標籤（`var(--text-xs)`）
- 當前路由對應項目以 `var(--color-accent)` 高亮
- `paddingBottom: safe-area-inset-bottom` 支援 iPhone 圓角螢幕

---

## 比較浮動列（CompareBar）

**顯示條件**：比較清單 ≥ 1 門課，且當前路由不是 `/compare`

| 元素 | 說明 |
|------|------|
| **浮動位置** | `position: fixed`，`bottom: {bottomOffset}`，`left: {sidebar-width 或 0}` |
| **bottomOffset** | 桌面版 `0`；行動版 `60`（底部導覽列高度，避免重疊） |
| **左邊距** | 桌面版 `var(--sidebar-width)`；行動版 `0`（無側邊欄） |
| **課程縮覽** | 已選課程名稱（最多 3 個）、數量文字 |
| **前往比較** | Primary 按鈕，點擊導至 `/compare` |
| **清除全部** | Ghost 按鈕，清空比較清單 |

## 全域 Toast 通知

使用 Sonner，統一處理以下事件：

| 事件 | 訊息範例 | 持續時間 |
|------|---------|---------|
| 加入最愛 | 「已加入最愛：{課程名稱}」 | 2s |
| 移除最愛 | 「已移除：{課程名稱}」 | 2s |
| 加入比較 | 「已加入比較清單」 | 2s |
| 比較清單已滿 | 「比較清單已達 3 門課上限」 | 3s |
| 資料載入失敗 | 「課程資料載入失敗，請稍後再試」 | 5s，含重試連結 |

## RWD 實作方式

**不使用 CSS media query**，改用 `useIsMobile(breakpoint = 768)` Hook 在 JavaScript 層判斷，確保與 inline-style 元件架構一致：

```typescript
// src/hooks/useIsMobile.ts
export function useIsMobile(breakpoint = 768): boolean {
  const [isMobile, setIsMobile] = useState(() =>
    typeof window !== 'undefined' ? window.innerWidth < breakpoint : false,
  );
  useEffect(() => {
    const mql = window.matchMedia(`(max-width: ${breakpoint - 1}px)`);
    const handler = (e: MediaQueryListEvent) => setIsMobile(e.matches);
    mql.addEventListener('change', handler);
    setIsMobile(mql.matches);
    return () => mql.removeEventListener('change', handler);
  }, [breakpoint]);
  return isMobile;
}
```

`Layout.tsx`、`Header.tsx`、`CourseListPage.tsx`、`CourseDetailModal.tsx`、`CompareBar.tsx` 均透過此 Hook 切換桌面/行動佈局。

## main 內容區 padding

| 情境 | paddingBottom |
|------|--------------|
| 桌面版（無比較浮動列） | 正常 |
| 桌面版（有比較浮動列） | CompareBar 高度（~56px） |
| 行動版（無比較浮動列） | 60px（底部導覽列） |
| 行動版（有比較浮動列） | 60px + CompareBar 高度 |

---

## 效能考量

- Header 使用 `position: sticky top: 0 z-index: 100`，不影響捲動效能
- CompareBar 條件渲染，避免不必要 DOM 掛載
- BottomNav 永遠掛載（行動版），避免路由切換時閃爍
