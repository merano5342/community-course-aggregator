## 需求背景

社大課程整合平台使用爬蟲抓取公共教育機構的課務資料，必須在明顯位置說明資料來源、更新頻率、使用限制與免責聲明，以符合透明度原則並降低被要求下架的風險。本頁同時作為使用者回報問題的入口。

## 各區塊說明

### 平台介紹

- **標題**：「社大課程整合平台是什麼？」
- **內文**：說明本平台目的（跨校整合瀏覽、比較、收藏），目標用戶（有意報名社大課程的市民）
- **版本資訊**：「目前版本：v{version}，最後更新：{lastUpdatedDate}」

### 資料來源

| 欄位 | 內容 |
|------|------|
| **資料提供者** | 各台北市社區大學，課務系統由 frog.tw 提供 |
| **抓取對象** | 僅抓取公開可見的課程列表與課程詳細資訊頁 |
| **存取方式** | 定期 HTTP 請求，不繞過任何驗證機制 |
| **資料性質** | 課程開課事實（名稱、時間、費用、老師、地點、狀態） |
| **不含內容** | 課程介紹全文、老師照片、學員個人資料 |
| **原始連結** | 每門課均附 `detailUrl`，指回原校課務系統 |

### 支援學校清單

動態根據 `data/courses/` 中實際存在的 JSON 產生：

| 欄位 | 說明 |
|------|------|
| **學校名稱** | 完整名稱 |
| **課務系統連結** | 原校 frog.tw 入口，以「前往原站」按鈕呈現 |
| **資料最後更新時間** | 讀自 JSON 的 `lastUpdated` 欄位（若有） |
| **課程總數** | 該校目前已抓取的課程數量 |

### 資料更新頻率

```
更新時間：每週一 00:00（UTC+0）自動更新
更新方式：GitHub Actions 觸發爬蟲，結果 commit 至 data/courses/ 資料夾
額滿狀態：僅供參考，實際以各校課務系統為準
非即時更新：本週內新增的課程或狀態變更，需等至下次排程才會反映
```

### 免責聲明

- 本平台為社群自主建置之非官方資訊整合工具，與各社大及 frog.tw 無任何官方合作
- 課程資訊（尤其是「名額狀態」）可能與原校系統有落差，報名前請以**原校官方課務系統**為準
- 本平台不提供任何報名功能，點擊「前往報名」將導向原校外部網站
- 本平台不對課程資訊的即時性、正確性負責

### 版權說明

- 課程資訊屬台北市各社大公開提供的公共教育資訊
- 本平台不主張對課程資料的所有權
- 如任何學校或機構對資料使用有疑慮，請透過「回報問題」聯繫，將立即移除對應學校資料
- 本平台程式碼以 MIT License 開源

### 回報問題

| 元素 | 說明 |
|------|------|
| **GitHub Issues 連結** | 「發現資料錯誤？課程資訊有誤？」→ 連結至 GitHub Issue 頁 |
| **回報按鈕** | 「回報問題」Primary 按鈕，新分頁開啟 GitHub Issues |
| **說明文字** | 「請在 Issue 中說明：哪所學校、哪門課程、哪個欄位有誤」 |

### 開源說明

| 元素 | 說明 |
|------|------|
| **GitHub 連結** | 「本平台為開源專案」+ GitHub Repo 連結 |
| **貢獻邀請** | 「歡迎 PR：新增更多社大、改善 UI、回報爬蟲 bug」 |
| **技術 Stack** | React + Vite + shadcn/ui + FullCalendar + Node.js + cheerio |

## 操作流程

使用者進入關於頁：
1. 滾動瀏覽各說明區塊
2. 點擊學校清單中「前往原站」連結，新分頁開啟原校課務系統
3. 點擊「回報問題」，新分頁開啟 GitHub Issues 並可直接建立 Issue
4. 點擊 GitHub Repo 連結，可查看原始碼或 star 專案

## 業務邏輯

- 支援學校清單與學校色彩 Map 使用與課程列表頁相同的資料來源，確保一致
- `lastUpdated` 時間從各校 JSON 頂層欄位讀取；若無此欄位，顯示「-」
- 頁面為純靜態資訊頁，無需任何 API 請求（學校清單除外）
