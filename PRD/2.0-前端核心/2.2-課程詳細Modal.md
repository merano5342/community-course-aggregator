# 課程詳細 Modal

---

**頁面位置**：覆蓋層（Overlay），從課程列表頁或我的最愛頁觸發  
**頁面路由**：無獨立路由（Overlay 模式）  
**所屬功能模組**：2.0 前端核心  

---

## 需求背景

課程詳細 Modal 提供使用者在不離開列表頁的情況下，查看單一課程的完整資訊，包含學費明細、老師介紹、地點、報名連結等。採用 Overlay 模式而非獨立頁面，保留使用者的篩選狀態，降低操作中斷感。

---

## 觸發方式

| 觸發位置 | 觸發動作 |
|---------|---------|
| 課程列表頁卡片 | 點擊卡片本體（非最愛/比較按鈕區域） |
| 我的最愛頁卡片 | 點擊卡片本體 |
| 時間表頁事件 | 點擊日曆上的課程事件 |

---

## Modal 佈局

```
┌─────────────────────────────────────────┐
│  [學校徽章]  [狀態 Badge]  [混成 Badge]  │  × 關閉
│                                         │
│  課程名稱（大字，多行）                  │
│  類別 > 子類別                           │
│                                         │
├─────────────────────────────────────────┤
│  資訊欄（左右兩欄 Grid）                 │
│  老師 │ 星期+時段                        │
│  費用 │ 開課日期                         │
│  地點 │ 週數                             │
│  名額狀態 │ 折扣資訊                     │
├─────────────────────────────────────────┤
│  （選填）老師簡介區塊                    │
├─────────────────────────────────────────┤
│  [加入最愛]  [加入比較]  [前往報名 →]   │
└─────────────────────────────────────────┘
```

行動版：全螢幕 Sheet（從底部滑入）。

## Modal 元素說明

### 頂部標識區

| 元素 | 說明 |
|------|------|
| **學校徽章** | 彩色 Badge，各校固定顏色，顯示學校短名 |
| **狀態 Badge** | 開放報名（綠）/ 名額已滿（紅）/ 已取消（灰）/ 延後（黃） |
| **新課程 Badge** | `isNew === true` 時顯示「NEW」 |
| **混成 Badge** | `isMixed === true` 時顯示「線上+實體」 |
| **關閉按鈕** | 右上角 ✕，或點擊 backdrop 關閉 |

### 課程資訊欄

| 欄位 | 資料來源 | 顯示格式 |
|------|---------|---------|
| **課程名稱** | `name` | 大字體，最多 3 行 |
| **類別** | `category` | 以 `>` 分隔層級，如「生活藝能 > 烹飪美食類」 |
| **授課老師** | `teacher[]` | 多人以「、」連接；若為空顯示「－」 |
| **上課星期** | `dayOfWeek` | 「每週{星期}」（日=0 轉換顯示） |
| **上課時段** | `timeSlot` | 上午 / 下午 / 晚上 |
| **學分費** | `fee` | NT$ {fee}；fee === 0 顯示「免費」 |
| **折扣** | `discount` | 若有折扣顯示「{discount}優惠」及劃線原價 |
| **開課日期** | `startDate` | {YYYY}/{MM}/{DD} 格式 |
| **總週數** | `weeks` | 共 {weeks} 週；若無此欄則不顯示 |
| **上課地點** | `location` | 完整地點名稱 |
| **名額狀態** | `status` | 文字說明（對應狀態 Badge 的文字版） |

### 操作按鈕區

| 按鈕 | 樣式 | 行為 |
|------|------|------|
| **加入最愛** | Secondary，已收藏時填色 | 切換收藏，Toast 通知 |
| **加入比較** | Secondary，已選時填色，清單滿時 disabled | 切換比較，Toast 通知 |
| **前往報名** | Primary + 外部連結圖示 | 新分頁開啟 `detailUrl`（原校課務系統） |

## 操作流程

### 開啟 Modal

1. 使用者點擊課程卡片本體
2. Backdrop 淡入，Modal 從下方滑入（或桌面版中央縮放出現）
3. Modal 顯示課程完整資訊
4. 焦點移入 Modal（無障礙 focus trap）

### 關閉 Modal

| 關閉方式 | 行為 |
|---------|------|
| 點擊 ✕ 按鈕 | 關閉 Modal，焦點回到觸發元素 |
| 點擊 Backdrop | 關閉 Modal |
| 按 Escape 鍵 | 關閉 Modal |

### 收藏動作

1. 使用者點擊「加入最愛」
2. Zustand favorites store 新增 course id
3. localStorage `cca_favorites` 更新
4. 按鈕立即變為「已加入最愛」（填滿愛心圖示）
5. Toast 通知「已加入最愛：{課程名稱}」
6. 使用者可繼續在 Modal 內操作，不自動關閉

### 前往報名動作

1. 使用者點擊「前往報名」
2. 新分頁開啟 `detailUrl`（外部：原校 frog.tw 課務系統）
3. Modal 保持開啟

---

## 業務邏輯

- Modal 為無路由 Overlay，不影響頁面 URL（不加 query param）
- 同一時間只能開啟一個 Modal
- 開啟 Modal 時背景內容禁止捲動（body overflow: hidden）
- `status === 'cancelled'` 時「前往報名」按鈕改為 disabled，tooltip 顯示「課程已取消」
- `status === 'full'` 時「前往報名」按鈕保持可點擊，但卡片顯示「名額已滿（仍可前往候補）」

---

## 錯誤處理

| 情境 | 處理方式 |
|------|---------|
| `detailUrl` 為空 | 「前往報名」按鈕 disabled，tooltip「暫無報名連結」 |
| 課程資料缺少必要欄位 | 欄位顯示「－」，不顯示錯誤 |

---

## 響應式設計

| 裝置 | Modal 樣式 |
|------|---------|
| 手機 < 768px | Bottom Sheet，滑入高度約 85vh，可下滑關閉 |
| 平板/桌機 ≥ 768px | 置中 Dialog，最大寬度 560px，backdrop 可點擊關閉 |

---

## 無障礙設計

- Modal 開啟時 `role="dialog"` + `aria-labelledby` 指向課程名稱
- Focus trap：Tab 鍵循環在 Modal 內部
- 關閉後焦點回到觸發的卡片
