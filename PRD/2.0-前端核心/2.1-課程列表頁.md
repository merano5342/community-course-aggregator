# 課程列表頁

---

**頁面位置**：首頁  
**頁面路由**：`/` - 社大課程整合平台  
**所屬功能模組**：2.0 前端核心  

---

## 需求背景

課程列表頁為本平台的主入口，使用者在此瀏覽、搜尋、篩選來自多所社大的課程，並快速決定是否收藏或加入比較清單。資料來源為 GitHub 靜態 JSON，透過 TanStack Query 載入並快取；所有篩選與排序皆在 client-side 完成，無需額外 API。

---

## 頁面佈局

```
[Header]
┌─────────────────────────────────────────────┐
│  搜尋框 + 結果統計                           │
├──────────────┬──────────────────────────────┤
│              │  排序選單  [卡片/列表 切換]   │
│  篩選側邊欄  ├──────────────────────────────┤
│  (桌面版)    │  課程卡片 Grid                │
│              │  （虛擬捲動 / 無限捲動）       │
│              ├──────────────────────────────┤
│              │  無結果空狀態（條件顯示）      │
└──────────────┴──────────────────────────────┘
[CompareBar（浮動，有選課時顯示）]
```

行動版：篩選側邊欄改為頂部「篩選」按鈕 → 開啟 Drawer。

---

## 搜尋框

| 元素 | 說明 |
|------|------|
| **輸入框** | placeholder：「搜尋課程名稱或老師…」，置頂固定於篩選欄上方 |
| **即時過濾** | keyup 後 debounce 200ms，對 `name` + `teacher[]` 欄位做 case-insensitive 比對 |
| **清除按鈕** | 輸入有值時顯示 ✕ 圖示，點擊清空並 refocus |
| **結果統計** | 搜尋框下方顯示：「共找到 {N} 門課」，篩選條件改變時即時更新 |

---

## 篩選側邊欄

### 篩選區塊清單

| 篩選項目 | 元件類型 | 說明 |
|---------|---------|------|
| **學校** | Checkbox 多選 | 列出所有已載入學校，含課程數量（如「南港社大（45）」） |
| **星期** | Toggle Button 多選 | 日～六 7 個按鈕，可多選 |
| **時段** | Toggle Button 多選 | 上午 / 下午 / 晚上，可多選 |
| **課程類別** | Accordion Tree | 一級分類展開，Checkbox 多選子項目 |
| **開課狀態** | Checkbox 多選 | 開放報名 / 名額已滿 / 取消 / 延後開課 |
| **折扣** | Checkbox 多選 | 無折扣 / 八折 / 五折… 依資料動態生成 |
| **僅顯示新課程** | Switch Toggle | 過濾 `isNew === true` |
| **僅顯示混成課程** | Switch Toggle | 過濾 `isMixed === true` |

### 篩選操作

| 操作 | 說明 |
|------|------|
| **即時過濾** | 任一篩選條件變更後立即重新計算結果，無需點擊「送出」 |
| **清除全部篩選** | 側邊欄頂部按鈕，重置所有篩選條件至預設 |
| **篩選條件計數** | 已套用篩選數量顯示於篩選按鈕旁（行動版） |
| **篩選條件持久化** | 條件存入 localStorage `cca_filters`，頁面重新整理後恢復 |

## 排序選單

| 排序選項 | 說明 |
|---------|------|
| 開課日期（近→遠） | 預設排序 |
| 開課日期（遠→近） | |
| 費用（低→高） | 依 `fee` ASC |
| 費用（高→低） | 依 `fee` DESC |
| 學校 A-Z | 依學校名稱排序 |

## 課程卡片

### 卡片元素

| 元素 | 說明 |
|------|------|
| **課程名稱** | 最多 2 行，超出截斷 |
| **學校徽章** | 彩色 Badge，各校固定顏色（如南港=藍、文山=綠） |
| **老師名稱** | 若多人以「、」連接，最多顯示 2 人 |
| **星期 + 時段** | 「每週三 晚上」格式 |
| **費用** | NT$ {fee}，若有折扣顯示劃線原價 + 折扣後價格 |
| **開課日期** | `{startDate}（共 {weeks} 週）` |
| **狀態 Badge** | 開放報名（綠）/ 名額已滿（紅）/ 已取消（灰）/ 延後（黃） |
| **新課程 Badge** | 若 `isNew` 顯示「NEW」標籤 |
| **混成 Badge** | 若 `isMixed` 顯示「混成」標籤 |
| **加入最愛按鈕** | 愛心圖示，已收藏時填滿，點擊切換 |
| **加入比較按鈕** | 比較圖示，已選時填滿，清單已滿時 disabled + tooltip |

### 卡片互動

| 互動 | 行為 |
|------|------|
| 點擊卡片本體 | 開啟課程詳細 Modal（2.2） |
| 點擊最愛按鈕 | 切換收藏，Toast 通知，不觸發 Modal |
| 點擊比較按鈕 | 切換比較清單，Toast 通知，不觸發 Modal |
| Hover 效果 | 卡片輕微上升 shadow，比較/最愛按鈕加深 |

## 虛擬捲動

- 使用 `@tanstack/react-virtual`（rowVirtualizer）
- 每頁估算高度 220px（卡片）
- 僅渲染視窗內 ± overscan 的卡片，確保 230+ 筆資料流暢捲動
- 篩選結果切換時重置捲動位置至頂部

---

## 空狀態

| 情境 | 說明 |
|------|------|
| **資料載入中** | 骨架屏（Skeleton）佔位，顯示 6 個卡片 placeholder |
| **載入失敗** | 錯誤訊息 + 重新整理按鈕 |
| **篩選無結果** | 插圖 + 「找不到符合條件的課程」+ 清除篩選按鈕 |
| **資料為空** | 「目前無課程資料，請等待每週更新」 |

## 資料載入

- 各校 JSON 路徑：`https://cdn.jsdelivr.net/gh/{repo}/data/courses/{school}.json`
- 多校並行 fetch（`Promise.all`），TanStack Query 各自快取，staleTime: 1 小時
- 首次載入完成前顯示骨架屏

---

## 操作流程

### 主要流程：瀏覽並收藏課程

1. 使用者開啟首頁，骨架屏顯示，背景並行載入各校 JSON
2. 載入完成後，顯示全部課程（預設依開課日期排序）
3. 使用者在搜尋框輸入關鍵字，列表即時過濾
4. 使用者勾選「星期三」+ 「晚上」篩選條件，列表即時縮減
5. 使用者點擊感興趣的卡片，開啟課程詳細 Modal（→ 2.2）
6. 在 Modal 內或直接在卡片上點擊愛心，加入最愛
7. Toast 顯示「已加入最愛」，可點擊連結前往 `/favorites`

### 加入比較流程

1. 使用者點擊卡片比較按鈕，CompareBar 從底部滑入
2. 最多選 3 門，第 4 門點擊時 Toast 警告「已達上限」並 disabled 按鈕
3. 點擊 CompareBar「前往比較」導至 `/compare`

## 業務邏輯

- 篩選邏輯為 AND 關係（同類型為 OR，不同類型為 AND）
  - 同學校多選：OR（南港 OR 文山）
  - 學校 AND 星期：AND（南港課程 AND 星期三）
- 費用篩選：`fee === 0` 歸類為「免費」，否則依折扣 badge 分類
- 搜尋字串拆字後分別對 `name` 與 `teacher[]` 做 includes 比對

---

## 效能考量

- 所有篩選/排序在 useMemo 內計算，避免不必要的重新計算
- 卡片圖片使用 `loading="lazy"` + 佔位背景色
- 搜尋 debounce 200ms，避免每個按鍵觸發計算
