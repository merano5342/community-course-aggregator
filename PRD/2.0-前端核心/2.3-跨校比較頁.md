# 跨校比較頁

---

**頁面位置**：跨校比較  
**頁面路由**：`/compare` - 跨校比較 | 社大課程整合平台  
**所屬功能模組**：2.0 前端核心  

---

## 需求背景

使用者在多所社大課程之間難以快速做出決策。跨校比較頁讓使用者最多選取 3 門課程並排呈現，並高亮差異欄位，協助快速判斷差異點（費用、時間、地點等）後直接前往報名。

## 比較表格

### 欄位列表（屬性列）

| 屬性 | 欄位對應 | 顯示格式 |
|------|---------|---------|
| **學校** | `school` | 學校全名 |
| **老師** | `teacher[]` | 以「、」連接 |
| **課程類別** | `category` | 完整類別字串 |
| **學分費** | `fee` | NT$ {fee}；0 顯示「免費」 |
| **折扣** | `discount` | 折扣值或「無折扣」 |
| **折後費用** | 計算值 | fee × discount 百分比 |
| **上課星期** | `dayOfWeek` | 週一 ～ 週日 |
| **上課時段** | `timeSlot` | 上午 / 下午 / 晚上 |
| **開課日期** | `startDate` | YYYY/MM/DD |
| **總週數** | `weeks` | {N} 週；無則顯示「－」 |
| **上課地點** | `location` | 完整地點 |
| **名額狀態** | `status` | 開放報名 / 名額已滿 / 已取消 / 延後 |
| **混成課程** | `isMixed` | 是 / 否 |
| **新課程** | `isNew` | 是 / 否 |

### 差異高亮規則

- 當某屬性列中，各欄的值**不完全相同**，該列背景高亮（淡黃色 / dark mode 淡橙色）
- 差異值本身以粗體顯示
- 不參與差異比較的屬性：課程名稱、老師（視為資訊欄）

## 課程欄標頭

每個課程欄的標頭顯示：

| 元素 | 說明 |
|------|------|
| **學校徽章** | 彩色 Badge |
| **狀態 Badge** | 開放報名 / 名額已滿 / ... |
| **課程名稱** | 最多 3 行，超出截斷 |
| **移除按鈕** | ✕ 圖示，點擊從比較清單移除此課程 |

## 操作流程

### 進入比較頁

1. 使用者在課程列表頁點擊 1–3 門課程的「比較」按鈕
2. CompareBar 顯示已選數量
3. 點擊 CompareBar「前往比較」按鈕，或點擊導覽列「比較」連結
4. 進入 `/compare`，顯示比較表格

### 新增課程

1. 當比較清單 < 3 門時，在空白欄位（或右側）顯示「+ 新增課程」按鈕
2. 點擊後導回 `/`（課程列表頁）
3. 列表頁保持篩選狀態，使用者繼續選課

### 移除課程

1. 點擊課程欄標頭的 ✕ 按鈕
2. 該欄從比較表移除，Zustand compareList 更新
3. 若清單歸零，顯示空狀態

### 前往報名

1. 點擊課程欄底部「前往報名」按鈕
2. 新分頁開啟 `detailUrl`
3. 比較頁保持不變

---

## 業務邏輯

- 比較清單儲存於 Zustand，**重新整理後清空**（不持久化至 localStorage）
- 最多 3 門課；第 4 門加入時在課程列表頁顯示 Toast 警告（不允許加入）
- 若某課程 `status === 'cancelled'`，「前往報名」按鈕 disabled，顯示「已取消」tooltip
- 比較表欄位若某課程該欄位為 null/undefined，顯示「－」
- 進入 `/compare` 但清單為空時，不顯示比較表，顯示引導空狀態

## 響應式設計

| 裝置 | 比較表呈現 |
|------|---------|
| 手機 < 768px | 橫向捲動（overflow-x: scroll），欄標頭固定置左 |
| 平板 768–1024px | 同上，但可顯示 2 欄 |
| 桌機 > 1024px | 全表可見，3 欄均衡分配寬度 |

固定左側屬性欄（sticky left）：使用 `position: sticky` 保持屬性名稱在橫向捲動時可見。
