# 我的最愛頁

---

**頁面位置**：我的最愛  
**頁面路由**：`/favorites` - 我的最愛 | 社大課程整合平台  
**所屬功能模組**：2.0 前端核心  

---

## 需求背景

使用者在瀏覽多校課程時，需要一個暫存區記錄有興趣的課程，在報名前整理候選清單。我的最愛頁提供：收藏清單管理、拖曳排序優先級、衝堂自動偵測、以及切換至週曆視圖確認時段衝突，協助使用者在截止前做出最終決策。

---

## 頁面元素

### 頂部區域

| 元素 | 說明 |
|------|------|
| **頁面標題** | 「我的最愛」 |
| **視圖切換** | Tab 或 Toggle：「清單視圖」/ 「週曆視圖」 |
| **課程計數** | 「共收藏 {N} 門課」 |
| **匯出按鈕** | 開啟匯出選項 Popover（詳見下方） |

### 衝堂警示橫幅

**顯示條件**：清單中有 ≥ 2 門課在同一（dayOfWeek + timeSlot）組合

| 元素 | 說明 |
|------|------|
| **警示圖示** | ⚠️ 橙色橫幅 |
| **衝堂說明** | 列出衝堂的時間段，如「週三晚上 有 2 門課衝堂」 |
| **快速跳轉** | 點擊可將捲動位置移至衝堂卡片 |

### 收藏卡片

| 元素 | 說明 |
|------|------|
| **拖曳把手** | 左側 `≡` 圖示，長按或點擊拖曳啟用 |
| **課程名稱** | 最多 1 行，超出截斷 |
| **學校徽章** | 彩色 Badge |
| **上課時段** | 星期 + 時段 |
| **衝堂標示** | 若此課與清單中其他課衝堂，顯示 🔴 紅色邊框 + 「衝堂」文字 |
| **老師名稱** | 簡短顯示 |
| **費用** | NT$ {fee} |
| **狀態 Badge** | 開放報名 / 名額已滿 |
| **移除按鈕** | 從最愛移除，確認後執行（或直接執行 + Undo Toast） |
| **加入比較按鈕** | 加入比較清單 |

## 拖曳排序

- 使用 `@dnd-kit/core` + `@dnd-kit/sortable`
- 長按（手機）或直接拖曳（桌面）拖曳把手 `≡`
- 拖曳時：卡片半透明 + 位置預覽線
- 放下後：順序立即更新，Zustand + localStorage 同步
- 鍵盤可用性：使用 @dnd-kit 內建 keyboard sensor（Space 選取，方向鍵移動）

---

## 衝堂偵測邏輯

```
衝堂定義：favorites 清單中，任意兩門課 dayOfWeek 相同 AND timeSlot 相同
```

| 步驟 | 說明 |
|------|------|
| 1 | 計算所有課程的（dayOfWeek, timeSlot）組合 |
| 2 | 若某組合出現 ≥ 2 次，標記為「衝堂組」 |
| 3 | 衝堂組內的所有課程卡片顯示紅框 + 衝堂標記 |
| 4 | 頂部橫幅列出所有衝堂時間段 |

## 匯出功能

點擊「匯出/分享」開啟 Popover，提供：

| 選項 | 說明 |
|------|------|
| **複製文字清單** | 格式：「{N}. {課程名稱}｜{學校}｜{星期}+{時段}｜NT${fee}」，逐行複製至剪貼簿 |
| **複製 Markdown 表格** | 同上但以 Markdown table 格式 |
| **截圖提示** | 建議使用瀏覽器截圖，平台不提供自動截圖 |

## 操作流程

### 整理候選課程

1. 使用者在課程列表頁將感興趣課程加入最愛
2. 進入 `/favorites` 查看完整清單
3. 若出現衝堂警示，使用者拖曳調整優先順序（想要的課排前面）
4. 點擊課程卡片本體，開啟課程詳細 Modal（→ 2.2）再次確認細節
5. 移除不需要的課程，直到無衝堂或接受衝堂狀態
6. 切換至「週曆視圖」直觀確認時段分布

### 移除課程

1. 點擊課程卡片右側「移除」按鈕
2. Toast 通知「已移除：{課程名稱}」，含 5 秒 **Undo（復原）** 按鈕
3. 5 秒內點擊 Undo：恢復原本位置
4. 5 秒後 Toast 消失：localStorage 永久移除

### 加入比較

1. 點擊課程卡片「比較」按鈕
2. 加入比較清單，CompareBar 從底部顯示
3. 可繼續在最愛頁操作，不跳轉

## 空狀態

| 情境 | 說明 |
|------|------|
| 清單為空 | 插圖 + 「還沒有收藏課程」 + 「前往瀏覽課程」按鈕 |
| 全部移除後 | 同上 |

## 業務邏輯

- 收藏資料存於 localStorage `cca_favorites`，格式：`string[]`（course id 陣列）
- 排序後的 id 順序同步寫入 localStorage
- 最大收藏數量：無硬性上限，但衝堂偵測以此清單為準
- course id 以 hash 自 URL query param `u=xxxxx` 生成，保證跨校唯一性
- 若某課程 JSON 資料已不存在（每週更新後下架），顯示「此課程已下架」卡片樣式，保留移除按鈕
