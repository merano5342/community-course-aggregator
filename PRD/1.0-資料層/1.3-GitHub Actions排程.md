# GitHub Actions 爬蟲排程

---

**所屬功能模組**：1.0 資料層  
**實作檔案**：`.github/workflows/scrape.yml`  
**建立日期**：2026-06-20

---

## 需求背景

爬蟲需每週自動執行，將最新課程資料寫入 `data/` 並推送至 `main`，觸發 GitHub Pages 重新部署。使用 GitHub Actions 的 `schedule` 觸發器實現，不需額外的 CI/CD 服務。

---

## 排程設定

```yaml
on:
  schedule:
    - cron: '0 16 * * 0'   # 每週日 UTC 16:00 = 台灣時間週一 00:00
  workflow_dispatch:         # 允許手動觸發
```

| 觸發方式 | 說明 |
|---------|------|
| **週期排程** | 每週執行一次（每週一凌晨台灣時間） |
| **手動觸發** | 透過 GitHub Actions UI 的「Run workflow」按鈕，供緊急更新或測試使用 |

---

## Workflow 步驟

```yaml
jobs:
  scrape:
    runs-on: ubuntu-latest
    permissions:
      contents: write    # 允許 git push

    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.12'
          cache: 'pip'

      - name: Install dependencies
        run: pip install -r scraper/requirements.txt

      - name: Run scraper
        run: python scraper/main.py
        env:
          SCHOOL_LIST: 'nangang,wenshan,songshan'

      - name: Check for changes
        id: diff
        run: |
          git diff --quiet data/ || echo "has_changes=true" >> $GITHUB_OUTPUT

      - name: Commit and push
        if: steps.diff.outputs.has_changes == 'true'
        run: |
          git config user.name  "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add data/
          git commit -m "chore(data): weekly course data update $(date -u +%Y-%m-%d)"
          git push
```

---

## 步驟說明

| 步驟 | 說明 |
|------|------|
| **Checkout** | 取得完整 repo（含 `data/` 舊版 JSON） |
| **Python setup** | 使用 pip cache 加速，固定 Python 3.12 |
| **Install deps** | 安裝 `scraper/requirements.txt` |
| **Run scraper** | 執行爬蟲，輸出寫入 `data/` |
| **Check diff** | `git diff data/` 若無變化則跳過 commit（避免空 commit） |
| **Commit & push** | 僅在有資料更新時才 commit，訊息含日期戳記 |

---

## 環境變數

| 變數名稱 | 說明 | 預設值 |
|---------|------|-------|
| `SCHOOL_LIST` | 逗號分隔的學校 id 清單 | `nangang,wenshan,songshan` |

機密資訊（如未來需要的 API key）存放於 GitHub Secrets，透過 `env:` 注入。

---

## 部署觸發鏈

```
GitHub Actions cron
  → scraper/main.py 執行
    → data/ JSON 更新
      → git push to main
        → GitHub Pages 自動重新部署（若設定 Pages from main/root）
          → jsDelivr CDN 快取更新（staleTime：24h）
```

前端透過 jsDelivr 取得資料：
```
https://cdn.jsdelivr.net/gh/{owner}/{repo}/data/courses/{school}.json
```

## 執行時間估算

| 動作 | 預估時間 |
|------|---------|
| Python 環境 + 安裝 | ~30s（有 pip cache 時） |
| 列表頁爬取（3 校） | ~10s |
| 詳細頁爬取（~300 門，並行 8） | ~2–5min |
| Git commit + push | ~10s |
| **總計** | **約 3–6 分鐘** |

---

## 失敗通知

GitHub Actions 預設在 workflow 失敗時發送 email 通知給 repo 擁有者。  
可額外設定 Slack/Discord webhook 通知（透過 `actions/github-script`）。

---

## 手動重跑

若某次排程失敗或資料有誤，可至 GitHub Actions 頁面：
1. 選取「Scrape Courses」workflow
2. 點擊「Run workflow」
3. 選擇分支（預設 `main`）並送出
