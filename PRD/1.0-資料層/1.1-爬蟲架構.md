## 需求背景

課程資料由各社大課務系統（frog.tw 平台）提供，無公開 API，需透過網頁爬蟲取得。爬蟲以 Python 撰寫，採用兩階段策略（列表頁 → 詳細頁），輸出靜態 JSON 供前端讀取。每週自動執行一次，以 GitHub Actions 排程驅動。

---

## 技術選型：Python

選 Python 而非 TypeScript/Node.js 的理由：

| 面向 | Python | Node.js |
|------|--------|---------|
| HTML 解析生態 | `BeautifulSoup4` 成熟穩定 | `cheerio` 可用，但生態略小 |
| 資料驗證 | `pydantic v2` 強型別、自動轉換 | `zod` 需手動序列化 |
| 非同步 HTTP | `aiohttp` 並行請求簡單 | `node-fetch` / `axios` 亦可 |
| Schema 同步 | Pydantic model → JSON Schema → TypeScript 可自動生成 | 需額外工具 |
| CI 相容性 | GitHub Actions 原生 Python 支援 | 同 |

## 兩階段爬取策略

### 第一階段：列表頁（Course List）

**目標**：取得當學期所有課程的 URL hash 及基本資訊。

- 網址格式：`https://{school}.frog.tw/course/m_course_list.php?mode=week&s={semester_hash}`
- 學期 hash 從列表頁 HTML 的 `<select id="q_semester">` 動態取得
- 頁面為**伺服器渲染靜態 HTML**（無需 Headless Browser）
- 每個課程格子的 tooltip 包含：課程名、開課日（dayOfWeek）、招生人數、報名人數、地點

從列表頁可取得：
- 課程 `u` hash（URL 參數）
- 課程名稱
- 星期（dayOfWeek）
- 招生人數（quota）
- 報名人數（enrolled）
- 地點（location）

### 第二階段：詳細頁（Course Detail）

**目標**：取得每門課的豐富資訊。

- 網址格式：`https://{school}.frog.tw/course/m_course_detail.php?u={hash}`
- 每頁亦為靜態 HTML，包含 Schema.org JSON-LD（可輔助解析）
- 主要擷取區塊：
  - `課前資訊`：適合對象（`targetAudience`）、上課工具、教學方式、評量方式、其他備註（`notes`）
  - `課程簡介`：理念 + 目標 → 合併為 `description`
  - `課程費用`：學分費（`fee`）、雜費
  - `課程大綱`：表格（週次 + 主題 + 內容）→ `outline[]`
  - `講師介紹`：姓名、現職、簡介（`teacherBio`）、專長、獎項

## 學期 Hash 取得邏輯

```python
async def get_semester_hash(school: str) -> str:
    url = f"https://{school}.frog.tw/course/m_course_list.php"
    html = await fetch(url)
    soup = BeautifulSoup(html, 'lxml')
    select = soup.find('select', id='q_semester')
    # 取第一個 option（最新學期）
    return select.find('option')['value']
```

## 並行請求策略

- 列表頁：學校數量少（≤5），同步循序即可
- 詳細頁：每校 100~300 門課，使用 `aiohttp` 並行，並設 semaphore 限流

```python
CONCURRENCY = 8   # 同時最多 8 個 requests
DELAY_BETWEEN = 0.5  # 每批次間隔 0.5 秒
```

## 輸出格式

每所學校輸出一個 JSON 檔案：

```
data/
├── courses/
│   ├── nangang.json    # Course[]
│   ├── wenshan.json
│   └── songshan.json
└── announcements/
    ├── nangang.json    # Announcement[]
    └── ...
```

- `data/courses/{school}.json`：Course 陣列（見 1.2-Course資料結構.md）
- 編碼：UTF-8，無 BOM
- 不輸出 `null`（optional 欄位若無值則省略 key）

## 錯誤處理

| 情境 | 處理方式 |
|------|---------|
| HTTP 錯誤（4xx/5xx） | 重試 3 次（指數退避），仍失敗則跳過該課程並記錄 warning |
| 解析失敗（缺欄位） | 以 `None` 填入 optional 欄位，必要欄位缺失則跳過並記錄 error |
| 學期 hash 取得失敗 | 整個學校跳過，不覆蓋舊 JSON |
| 輸出 JSON 與舊版完全相同 | 不觸發 git commit（由 output.py 比對 diff） |
