## 需求背景

課程資料由各社大課務系統（frog.tw 平台）提供，無公開 API，需透過網頁爬蟲取得。爬蟲以 Python 撰寫，採用兩階段策略（列表頁 → 詳細頁），輸出靜態 JSON 供前端讀取。每週自動執行一次，以 GitHub Actions 排程驅動。

---

## 技術選型：Python

選 Python 而非 TypeScript/Node.js 的理由：

| 面向 | Python | Node.js |
|------|--------|---------|
| HTML 解析生態 | `BeautifulSoup4` 成熟穩定 | `cheerio` 可用，但生態略小 |
| 資料驗證 | `pydantic v2` 強型別、自動轉換 | `zod` 需手動序列化 |
| 非同步 HTTP | `aiohttp` 並行請求簡單 | `node-fetch` / `axios` 亦可 |
| Schema 同步 | Pydantic model → JSON Schema → TypeScript 可自動生成 | 需額外工具 |
| CI 相容性 | GitHub Actions 原生 Python 支援 | 同 |

## 爬取策略：增量爬（Incremental）

**決策**：採用方案 B——增量爬取詳細頁。

| 方案 | 速度 | 伺服器壓力 | 說明 |
|------|------|---------|------|
| A. 全量爬 | ~10 分鐘 | 較大 | 每次爬所有 detail page |
| **B. 增量爬** | **~2 分鐘** | **友善** | **只爬新增或異動課程的 detail page（已選）** |
| C. 僅爬 list | < 1 分鐘 | 最友善 | 犧牲 outline/teacherBio |

### 增量邏輯

```
每週執行時：
1. 爬列表頁 → 得到本學期所有課程的 { u_hash, quota, enrolled, ... }
2. 讀取現有 data/courses/{school}.json（舊資料）
3. 比對差異：
   a. 新 hash（舊 JSON 沒有）→ 必須爬 detail page
   b. hash 存在 + quota/enrolled 有變化 → 只更新這兩個欄位，不重爬 detail
   c. hash 存在 + 無變化 → 直接沿用舊資料
4. 並行爬取「新課程」的 detail pages
5. 合併輸出新 JSON
```

**關鍵觀察**：
- `quota` / `enrolled` 在 **list page 的 tooltip** 就有，每週都更新，不需進 detail page
- `description` / `outline` / `teacherBio` 學期開始後幾乎不變，只需在課程**首次出現**時爬一次
- 新課程主要在**換學期**時集中出現（每年兩次），其餘週僅個別異動

## 兩階段爬取流程

### 第一階段：列表頁（每週必執行）

**目標**：取得本學期所有課程的 hash + 即時數字。

- 網址：`https://{base_url}/course/m_course_list.php?mode=week&s={semester_hash}`
- 學期 hash 從 `<select id="q_semester">` 的第一個 `<option>` 取得（最新學期）
- 頁面為**伺服器渲染靜態 HTML**，無需 Headless Browser
- 每個課程格子的 tooltip 含：課程名、星期、招生人數、報名人數、地點

從列表頁更新的欄位（每週）：
- `quota`（招生人數）
- `enrolled`（報名人數）
- `status`（由 quota/enrolled 推算）

### 第二階段：詳細頁（僅限新課程）

**目標**：首次出現的課程，補齊豐富資訊。

- 網址：`https://{base_url}/course/m_course_detail.php?u={hash}`
- 靜態 HTML，含 Schema.org JSON-LD
- 擷取區塊：
  - 課前資訊 → `targetAudience`、`notes`
  - 課程簡介 → `description`（理念 + 目標合併）
  - 課程費用 → `fee`
  - 課程大綱 → `outline[]`（週次 + 主題）
  - 講師介紹 → `teacherBio`

## 學期 Hash 取得邏輯

```python
SCHOOL_BASES = {
    'nangang':    'https://nangang.frog.tw',
    'songshan':   'https://ss.twcc.org.tw',
    'wenshan':    'https://wenshan.wenshan.org.tw',
    'beitou':     'https://bt.btcc.org.tw',
    'wanhua':     'https://whcc.twcu.org.tw',
    'xinyi':      'https://xy.twcu.org.tw',
    'zhongzheng': 'https://zzcc.twcc.org.tw',
    'zhongshan':  'https://zscc.twcu.org.tw',
}

async def get_semester_hash(base_url: str) -> str:
    url = f"{base_url}/course/m_course_list.php"
    html = await fetch(url)
    soup = BeautifulSoup(html, 'lxml')
    select = soup.find('select', id='q_semester')
    # 取第一個 option（最新學期）
    return select.find('option')['value']
```

## 輸出格式

每所學校輸出一個 JSON 檔案：

```
data/
├── courses/
│   ├── nangang.json    # Course[]
│   ├── wenshan.json
│   └── songshan.json
└── announcements/
    ├── nangang.json    # Announcement[]
    └── ...
```

- `data/courses/{school}.json`：Course 陣列（見 1.2-Course資料結構.md）
- 編碼：UTF-8，無 BOM
- 不輸出 `null`（optional 欄位若無值則省略 key）

## 錯誤處理

| 情境 | 處理方式 |
|------|---------|
| HTTP 錯誤（4xx/5xx） | 重試 3 次（指數退避），仍失敗則跳過該課程並記錄 warning |
| 解析失敗（缺欄位） | 以 `None` 填入 optional 欄位，必要欄位缺失則跳過並記錄 error |
| 學期 hash 取得失敗 | 整個學校跳過，不覆蓋舊 JSON |
| 輸出 JSON 與舊版完全相同 | 不觸發 git commit（由 output.py 比對 diff） |
