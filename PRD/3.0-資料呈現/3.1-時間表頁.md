# 時間表頁

---

**頁面位置**：時間表  
**頁面路由**：`/schedule` - 時間表 | 社大課程整合平台  
**所屬功能模組**：3.0 資料呈現  

---

## 需求背景

使用者在整理候選課程後，需要直覺地看出課程在哪些時間段分布、是否與自身行程衝突。時間表頁以 FullCalendar 週視圖呈現「我的最愛」清單中的所有課程，色彩區分學校，衝堂區塊視覺重疊，輔助使用者做出最終報名決策。

---

## 頁面元素

### 導航控制列

| 元素 | 說明 |
|------|------|
| **上/下週按鈕** | `<` / `>` 箭頭，切換當前顯示週 |
| **今天按鈕** | 返回包含今日的週，讓使用者快速定位 |
| **日期範圍顯示** | 「YYYY年 M月 D日 – M月 D日」格式 |
| **視圖切換** | 週視圖（預設）/ 月視圖 切換（月視圖顯示每日有無課程的點標記） |

### 日曆事件（課程區塊）

| 元素 | 說明 |
|------|------|
| **事件背景色** | 依學校固定顏色：南港（藍）/ 文山（綠）/ 松山（橙）/ 其他學校（灰）… |
| **事件文字** | 課程簡短名稱（最多 2 行） |
| **衝堂視覺** | 同時段 2 門課：並排各佔 50% 寬，3 門各 33%；邊框加深 + 衝堂警示圖示 |
| **點擊事件** | 開啟課程詳細 Modal（→ 2.2） |

### 時段定義

FullCalendar 的時間軸對應課程 `timeSlot` 欄位：

| timeSlot | 日曆時間範圍 | 顯示標籤 |
|----------|------------|---------|
| `morning` | 09:00 – 12:00 | 上午 |
| `afternoon` | 13:00 – 17:00 | 下午 |
| `evening` | 18:30 – 21:30 | 晚上 |

課程事件佔滿對應時段格（非精確到分鐘，視 frog.tw 資料而定）。

### 圖例列

- 位於日曆底部，顯示各學校對應的色彩方塊 + 學校名稱
- 依目前最愛清單中實際出現的學校動態生成（不含未收藏的學校）

### 衝堂提示列

**顯示條件**：最愛清單中有衝堂課程

| 元素 | 說明 |
|------|------|
| **警示橫幅** | 「⚠️ 本週有 {N} 個衝堂時段，請確認後再報名」 |
| **衝堂列表** | 列出「{星期}+{時段}：{課程A} 與 {課程B} 衝堂」 |

## 操作流程

### 查看時間分布

1. 使用者在「我的最愛」頁點擊「週曆視圖」Tab，或直接進入 `/schedule`
2. FullCalendar 載入，顯示目前週（包含今日）
3. 最愛清單中的所有課程顯示為對應時段的彩色區塊
4. 使用者一眼看出哪幾天有課、哪天衝堂

### 確認衝堂

1. 衝堂課程在同一日曆格並排顯示，邊框加橙色警示
2. 底部衝堂提示列說明具體衝堂課程
3. 點擊任一衝堂課程，開啟 Modal 查看詳細資訊後決定是否移除

### 瀏覽其他週（選修開課時段橫跨不同週）

1. 使用者點擊 `>` 切換至下週
2. 日曆更新，事件仍顯示（課程每週重複，FullCalendar 依 `dayOfWeek` 展開重複事件）

## FullCalendar 設定

| 設定項 | 值 | 說明 |
|------|------|------|
| `initialView` | `timeGridWeek` | 預設週視圖 |
| `headerToolbar` | 自訂（使用上述導航控制列替換） | |
| `allDaySlot` | `false` | 隱藏全天事件列 |
| `slotMinTime` | `'08:00:00'` | 日曆從早上 8 點開始 |
| `slotMaxTime` | `'22:00:00'` | 日曆到晚上 10 點結束 |
| `locale` | `'zh-tw'` | 中文星期標籤 |
| `eventClick` | 開啟課程 Modal | |
| `eventContent` | 自訂渲染（顯示課程名稱 + 狀態 Badge） | |

## 資料轉換

最愛清單的 Course 物件需轉換為 FullCalendar Event 物件：

```typescript
// 轉換邏輯（概念）
const timeSlotMap = {
  morning:   { start: '09:00', end: '12:00' },
  afternoon: { start: '13:00', end: '17:00' },
  evening:   { start: '18:30', end: '21:30' },
};

// 轉為每週重複事件（rrule 或手動指定 daysOfWeek）
{
  title: course.name,
  daysOfWeek: [course.dayOfWeek], // 0=日 ~ 6=六
  startTime: timeSlotMap[course.timeSlot].start,
  endTime: timeSlotMap[course.timeSlot].end,
  startRecur: course.startDate,
  backgroundColor: schoolColorMap[course.school],
  extendedProps: { courseId: course.id },
}
```

## 空狀態

| 情境 | 說明 |
|------|------|
| 最愛清單為空 | 「還沒有收藏課程，前往課程列表加入最愛」+ 連結按鈕 |
| 當前週無課程（使用者翻到沒有開課的週） | 日曆正常顯示，事件為空，不另行提示 |

## 響應式設計

| 裝置 | 呈現方式 |
|------|---------|
| 手機 < 768px | 週視圖水平捲動（僅顯示 3 天，左右滑動切換）；或改用 listWeek 視圖（清單式） |
| 桌機 ≥ 768px | 完整 7 天週視圖 |

行動版 listWeek 視圖：以清單列出當週各天的課程，比直式日曆更易讀。

---

## 業務邏輯

- 時間表頁僅展示「我的最愛」清單中的課程，不展示全部課程
- 事件依學校固定顏色區分，顏色 Map 與課程列表頁卡片的學校 Badge 顏色一致
- 課程為每週重複事件（`startRecur` 設為開課日，`endRecur` 依 weeks 計算）
- 若課程 `status === 'cancelled'` 或 `status === 'full'`，事件仍顯示但加灰色蒙層
